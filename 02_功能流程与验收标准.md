# 02. 功能流程与验收标准

版本：v2.0  
日期：2026-05-22

## 1. 说明

本文件是全流程摘要。每条流程的详细开发内容、用户体感、依赖和周期，业务方建议查看 `flow-pages/` 下对应网页文档，开发可查看 `flows/` 下对应Markdown文档。

| 编号 | 流程 | 目的 | 核心验收 |
| --- | --- | --- | --- |
| F01 | 课程发送与预约下单闭环 | 让妆教师能从OPC选择课程链接转发给客户，客户完成选老师、选时间、已付款/未付款预约，老师和老板同步看到预约状态。 | 客户可从课程链接进入预约页并完成预约；已付款和未付款两条路径都能跑通；预约成功后客户和老师均收到状态反馈；预约单状态与详情一致 |
| F02 | 课程新增与修改闭环 | 让妆教师维护自己的课程，老板可统一查看课程设置能力。 | 可新增课程并上架；有预约课程修改时出现影响提示；下架课程不会影响历史订单；表单校验完整 |
| F03 | 课前问卷与素颜面诊报告闭环 | 客户课前填写问卷并上传素颜照片，系统生成客户版和专业版报告，妆教师据此准备课程方案。 | 客户可提交问卷和照片；客户版报告话术友好且不改变骨相；专业版报告包含五官、肤质、风格、方案；妆教师可确认课前课程方案 |
| F04 | 到店上课与课程小结发送闭环 | 妆教师到店开始上课、记录课程重点、生成课程小结，并选择是否连同产品推荐一起发送。 | 老师可开始并结束课程；课程小结可编辑确认；产品推荐可修改后发送；客户可查看学习回顾 |
| F05 | Cosark商城购买与订单佣金同步闭环 | OPC展示商品推荐，交易在Cosark完成，订单和佣金同步回OPC。 | 商品能从OPC跳转到Cosark；订单支付后可同步回OPC；佣金状态准确；退款后佣金回滚 |
| F06 | BeautyClaw课程资产与专属妆教师绑定闭环 | 客户课程资产在BeautyClaw管理，首次约课绑定专属妆教师，老板可处理离职继承。 | 客户在BeautyClaw查看课程资产；首次约课后默认绑定老师；老板可完成客户继承；妆教师只能看到自己的客户 |
| F07 | 订单取消与改期闭环 | 客户或妆教师可发起取消/改期，并通过审批确认结果。 | 客户可提交取消或改期；老师可审批；处理结果同步给客户；历史记录可追溯 |
| F08 | 团队与妆教师管理闭环 | 老板统一管理团队妆教师，包含邀请、新增、编辑权限、解绑、离职处理。 | 老板可邀请/新增/编辑/解绑老师；离职前提示客户继承；妆教师端不可见老板管理工具 |
| F09 | 团队模板配置闭环 | 老板统一维护问卷等模板，妆教师只读调用。 | 老板可编辑并发布模板；发布影响有确认；妆教师不可编辑模板；复制问卷链接可用 |
| F10 | 定时任务与自动触达闭环 | 预约后自动生成课前/课后触达任务，并可人工确认或补发。 | 任务按规则生成；可调整发送时间；失败任务可补发；客户可点击卡片进入对应页面 |
| F11 | 私域助手与客服托管闭环 | 妆教师上课时可让AI辅助回复企微客户，并按关键词转人工。 | 三种托管模式可切换；半托管需老师确认；关键词触发转人工；回复记录可追溯 |
| F12 | 桌面端运营后台支撑闭环 | 支撑OPC平台审核、通知、配置、异常处理和经营数据查看。 | 后台可审核用户；可发布通知；可处理异常；配置可影响前台规则 |
| F13 | 算力管理与充值消耗闭环 | 为已开发AIGC提供算力充值、扣减、明细和失败返还账本。 | 支付后算力到账；AIGC使用正确扣减；失败自动返还；明细可按类型和日期查询 |
| F14 | 选品中心与分销卡片生成闭环 | OPC用户可从Cosark商品池选品，生成可转发的分销卡片。 | 可筛选商品；商品详情显示佣金；可生成卡片链接；下架商品不可继续推荐 |
| F15 | 分销订单与收益结算闭环 | 展示通过OPC产生的分销订单、佣金、售后回滚和提现结算。 | 订单状态同步准确；佣金金额正确；退款可回滚；提现申请可追踪 |
| F16 | 邀请朋友奖励与推广关系闭环 | OPC用户可邀请他人加入平台，并按审核通过等规则获得邀请奖励。 | 邀请链接带归因；被邀请人审核通过后生成奖励；奖励状态可追踪；与团队佣金分离 |
| F17 | 通知中心与业务跳转闭环 | 统一承载审核、订单、系统、算力等通知，并能跳转对应业务。 | 后台可发布通知；用户可接收并查看详情；通知可跳转对应业务；已读状态正确 |
| F18 | 账号互通与跨小程序授权闭环 | 打通BeautyOPC、BeautyClaw、Cosark之间的账号与授权关系。 | 三端账号绑定状态清晰；授权范围明确；手机号异常可处理；解绑有风险提示 |
| F19 | 账户设置与帮助支持闭环 | 补齐账户设置、使用教程、联系客服、用户协议等基础页。 | 基础页不为空；资料可保存；反馈可生成工单；协议可阅读确认 |
| F20 | 接单广场机会浏览与申请占位闭环 | 让首页接单广场从静态展示变成有字段、有详情、有待开放边界的机会入口。 | 接单广场不再是空态；机会详情字段清楚；申请功能边界明确；收藏后有反馈 |
