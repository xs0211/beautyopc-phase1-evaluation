# 01. BeautyOPC 一期范围冻结说明

版本：v1.0  
日期：2026-05-20

## 1. 一期冻结结论

BeautyOPC 一期冻结为：

> BeautyOPC 通用平台底座 + AIGC 内容生成与算力 + 妆造私教样板行业 + 商城分销互通 + 桌面运营后台。

一期不追求一次性完成完整生态，而是先完成可试点、可演示、可运营的核心闭环。

## 2. 一期必须解决的核心问题

| 问题 | 一期解决方式 |
| --- | --- |
| 用户不知道如何加入平台 | 做 OPC 入驻和后台审核 |
| 用户不会持续生产内容 | 接入 AIGC 内容生成入口、AIGC 算力和素材记录 |
| 用户缺少赚钱机会 | 做接单广场和选品分销入口 |
| 平台无法真实运营 | 做桌面运营后台 |
| 妆造师需求已经明确 | 用妆造私教作为样板行业跑通闭环 |
| 团队老板需要管理经营 | 做老板看板、老师管理、客户继承和团队数据 |

## 3. 一期 P0 范围

### 3.1 BeautyOPC 小程序底座

必须完成：

- 微信登录。
- 用户基础资料。
- OPC 入驻申请。
- 入驻审核状态展示。
- 首页接单广场入口。
- AIGC 内容生成入口。
- 选品中心入口。
- 运营中心入口。
- 通知中心。
- 我的页面基础能力。
- 算力管理入口，仅用于 AIGC 内容生成。
- 分销订单入口。
- 邀请奖励入口和基础记录。
- 基础内容页：使用教程、联系客服、账户设置、用户协议、隐私政策。

不在一期做重：

- 复杂会员体系。
- 多级用户成长体系。
- 完整内容社区。
- 完整私域托管助手。

### 3.2 OPC 入驻与审核

必须完成：

- 用户提交入驻资料。
- 后台查看资料。
- 后台通过或驳回。
- 驳回原因展示。
- 用户端显示审核中、已通过、已驳回。

一期不做：

- 多节点审批流。
- 自动审核评分。
- 复杂资质认证。

### 3.3 接单广场

必须完成：

- 后台发布任务。
- 任务列表。
- 任务详情。
- 任务筛选。
- 用户报名或接单。
- 我的任务。
- 后台查看报名人。
- 任务基础状态。

一期策略：

- 任务由平台后台代商家或品牌方发布。
- 商家自助发任务放到后续版本。

一期不做：

- 商家自助后台。
- 复杂任务竞价。
- 复杂任务结算和自动打款。
- 多轮交付验收。

### 3.4 AIGC 内容生成、算力与素材记录

必须完成：

- BeautyOPC 进入 AIGC 内容生成入口。
- 图文生成、图片生成、视频生成、视频混剪等内容生成能力入口，按原 BeautyOPC AIGC 设计稿承接。
- AIGC 算力余额展示。
- AIGC 算力消耗记录。
- 生成记录或素材库基础能力。

一期不重复开发：

- AI 生成模型能力本体。
- AI 测肤、AI 推品、AI 妆效等妆造运营小工具不放在 AIGC 模块，不消耗 AIGC 算力。
- BeautyOPC 与 BeautyClaw 不做统一算力账本打通。

### 3.5 AIGC 算力充值与账本

必须完成：

- 算力余额。
- 算力套餐。
- 微信支付充值。
- 支付回调。
- 充值订单。
- 消耗记录。
- 算力仅应用于 BeautyOPC AIGC 内容生成模块。

一期必须重点确认：

- 算力账本归属。
- AIGC 任务扣减发起方。
- 失败回滚。
- 退款规则。
- 重复支付回调处理。

### 3.6 选品中心与商城分销

必须完成：

- BeautyOPC 展示可推广商品。
- 商品价格、佣金、推荐信息展示。
- 跳转商城商品详情。
- 跳转时携带 OPC 推广人参数。
- 商城成交后可查询分销订单。
- BeautyOPC 展示分销订单和收益概览。
- 提现入口跳转商城或展示提现状态。

系统边界：

- 商城负责商品详情、支付、订单、售后、佣金计算、提现打款。
- BeautyOPC 负责商品入口、推广归因参数、订单和收益展示。

### 3.7 邀请奖励

必须完成基础闭环：

- 生成邀请链接或邀请码。
- 被邀请人注册时绑定邀请关系。
- 展示邀请记录。
- 展示奖励状态。

一期暂不做重：

- 多级邀请。
- 复杂邀请佣金自动结算。
- 邀请奖励自动提现。

备注：

- 商品分销佣金归商城体系。
- 邀请其他 OPC 或妆造师加入平台产生的邀请奖励，需要与商品佣金分开统计。

### 3.8 妆造私教样板行业

必须完成：

- 妆造师工作台。
- 老板工作台。
- 课程设置。
- 课程链接转发。
- 客户预约。
- 课程和授课老师选择。
- 已付款客户上传凭证或核销码。
- 未付款客户付款或进入支付路径。
- 可预约时间展示。
- 预约成功提醒。
- 课前问卷。
- 素颜面诊照片上传。
- AI 客户版报告。
- AI 专业版报告。
- 客户档案。
- 到店开始上课。
- 课堂记录或录音状态。
- AI 课程总结。
- 妆造师确认后发送总结。
- 产品推荐生成。
- 总结与产品推荐可合并发送或分开发送。
- 商品购买按钮跳转商城。
- 老板管理老师。
- 客户绑定老师。
- 客户继承给其他老师。
- 团队统一问卷模板。
- 运营工具分组：妆造师定制工具、通用工具、管理工具。
- AI 测肤、AI 面部分析、AI 妆效生成、AI 推品等放在运营通用工具。
- 管理工具仅老板可见，妆造师不可见。

一期不做重：

- 多门店复杂结算。
- 复杂排班审批。
- 复杂模板自由编辑器。
- 完整客服托管机器人。

### 3.9 桌面运营后台

必须完成：

- 后台登录。
- 用户管理。
- OPC 审核。
- 任务管理。
- 通知管理。
- 内容管理。
- AIGC 算力管理。
- 分销订单查询。
- 邀请奖励查询。
- 妆造业务管理。
- 老板和妆造师管理。
- 基础数据看板。
- 后台角色权限基础版。

一期不做：

- 复杂 BI 报表。
- 商家自助后台。
- 园区/政府合作后台。
- 开放平台和服务商管理。

## 4. 一期明确不做范围

以下能力进入二期或后续版本，不纳入一期开发主承诺：

| 能力 | 后置原因 |
| --- | --- |
| 私域助手全托管/半托管/不托管完整能力 | 涉及微信或企业微信消息权限、AI安全、执行日志，风险高 |
| AI 自动获客 | 涉及外部平台数据、内容风控和获客链路 |
| 热点中心完整榜单 | 可作为内容创作增强，不影响一期核心闭环 |
| 商家自助发布任务 | 一期先由平台后台代发，降低复杂度 |
| 多行业模板 | 一期先用妆造私教验证样板 |
| 复杂佣金结算和自动打款 | 商品佣金先依赖商城，邀请奖励先做基础记录 |
| 多门店复杂组织架构 | 一期只做团队老板和妆造师基础关系 |
| 完整课程中心和培训体系 | 可在后续作为增长期能力 |

## 5. 一期角色权限冻结

| 角色 | 一期可见内容 |
| --- | --- |
| OPC 用户 | 自己的入驻、任务、AIGC、AIGC 算力、选品、分销、邀请奖励 |
| 妆造师 | 自己的课程、预约、客户、报告、课程总结、销售额、任务 |
| 团队老板 | 妆造师全部能力 + 团队经营、老师管理、客户继承、团队佣金、统一模板 |
| 客户 | 自己的课程预约、订单、问卷、面诊报告、课程总结、产品推荐 |
| 平台运营 | 全平台用户、任务、通知、AIGC 内容与算力、分销、邀请、妆造业务 |

关键原则：

- 老板端只做增加，不做删减。
- 妆造师不能看到团队佣金总账。
- 妆造师可以使用通用 AI 小工具，但不能看到老板管理工具。
- AIGC 算力不与 BeautyClaw 共用。
- 客户端不显示客服托管状态。
- 客户看到的是微信聊天链接和小程序服务页，不进入妆造师工作台。
