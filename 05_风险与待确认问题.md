# 05. 风险与待确认问题

版本：v1.0  
日期：2026-05-20

## 1. 高优先级风险

| 风险 | 影响 | 建议处理 |
| --- | --- | --- |
| 商城账号与推广归因无法稳定互通 | 选品、分销、产品推荐订单归因受影响 | 第 1 周确认 unionid/openid/统一用户 ID 与跳转参数 |
| AIGC 算力账本规则不清楚 | 可能出现扣减错误、不到账、用户投诉 | 第 1 周确认账本归属、扣减方、回滚和退款规则 |
| 商城不能识别 BeautyOPC 推广人 | 分销闭环无法成立 | 第 1 周完成跳转参数和归因测试 |
| 商城不能回传分销订单 | BeautyOPC 无法展示收益 | 第 1 周确认订单和佣金接口 |
| 妆造课程是否真实支付未确认 | 预约链路和订单状态会变复杂 | 第 1 周确认真实支付或凭证核销方案 |
| AI 报告生成能力不确定 | 妆造样板演示效果受影响 | 第 2 周确认模板生成还是模型生成 |
| 后台范围过大 | 排期膨胀 | 一期只做运营必需能力 |
| 私域助手被提前拉入一期 | 会显著增加企业微信/微信消息风险 | 明确后置到二期或专项评估 |

## 2. 开发前必须确认的问题

### 2.1 账号与主体

1. BeautyOPC 与商城是否同主体？
2. 是否绑定同一个微信开放平台？
3. 是否可以稳定获取 unionid？
4. 现有用户系统是否已有统一用户 ID？
5. 用户手机号是否必须授权？

### 2.2 AIGC 与运营通用 AI 小工具

1. AIGC 模块一期确认包含哪些能力：图文生成、图片生成、视频生成、视频混剪？
2. 每个 AIGC 能力由哪个服务提供？
3. AIGC 算力余额查询由哪个系统提供？
4. AIGC 算力扣减由 BeautyOPC 发起还是 AIGC 服务发起？
5. AIGC 生成结果保存在哪个系统？
6. 是否已有素材库？
7. AI 测肤、AI 面部分析、AI 妆效生成、AI 推品是否作为运营通用工具入口提供？
8. 面诊报告、课程总结、产品推荐使用模型生成还是模板生成？
9. BeautyClaw 是否仅保留为后续外部跳转，不进入一期算力体系？

### 2.3 商城

1. 商城是否已有商品列表接口？
2. 是否支持外部小程序跳转商品详情？
3. 推广人参数格式是什么？
4. 是否支持客户绑定妆造师后的长期归因？
5. 分销订单接口是否可查询？
6. 佣金余额和提现状态是否可查询？
7. 退款后佣金状态如何变化？
8. 商品推荐是否需要指定商品库？

### 2.4 微信支付

1. AIGC 算力充值使用哪个商户号？
2. 支付回调由哪个后端接收？
3. 充值订单在 BeautyOPC 还是统一支付系统管理？
4. 妆造课程未付款路径是否需要真实支付？
5. 如果课程使用真实支付，收款主体是谁？
6. 是否需要退款？

### 2.5 妆造样板业务

1. 课程类型、价格、时长、样板是否最终确认？
2. 可预约时间规则是否最终确认？
3. 已付款凭证和核销码两种方式是否都做？
4. 问卷模板字段是否最终确认？
5. 面诊报告的专业版和客户版字段是否最终确认？
6. 产品推荐商品池是否来自商城？
7. 产品推荐是否必须包含工具和底妆产品？
8. 老板是否可修改团队统一问卷模板？
9. 妆造师是否完全不能修改团队模板？

### 2.6 后台

1. 是否已有后台技术框架？
2. 后台账号角色有哪些？
3. 平台运营和团队老板是否在同一后台？
4. 是否需要操作日志？
5. 数据导出是否一期需要？
6. 后台是否需要移动端适配？

## 3. 建议决策清单

| 决策项 | 建议默认方案 | 决策时间 |
| --- | --- | --- |
| 商家自助发任务 | 一期不做，后台代发 | 第 1 周 |
| 私域助手 | 一期不做完整托管，只保留入口或规划 | 第 1 周 |
| 课程支付 | 若支付接口成熟则做真实支付，否则先凭证/核销码 | 第 1 周 |
| AI 报告 | 一期可模板化生成，保留模型增强接口 | 第 2 周 |
| 素材库 | AIGC 生成结果至少在 BeautyOPC 保留基础记录 | 第 1 周 |
| 邀请奖励 | 一期做关系和状态，复杂结算后置 | 第 1 周 |
| 老板后台 | 一期在 BeautyOPC 工作台 + 平台后台支撑，不做独立老板后台 | 第 2 周 |
| 数据看板 | 一期只做核心经营数据，不做复杂 BI | 第 2 周 |

## 5. 验收风险提醒

开发验收时必须做真实链路测试：

- 新用户注册到审核通过。
- 用户充值 AIGC 算力到 AIGC 扣减。
- BeautyOPC 完成 AIGC 生成后返回记录。
- BeautyOPC 跳转商城后完成推广归因。
- 商城订单回传分销数据。
- 妆造师发课程链接到客户完成预约。
- 客户完成问卷和面诊。
- 妆造师查看报告并发送总结和推荐。
- 老板管理老师和客户继承。
- 后台处理审核、任务、通知、AIGC 内容与算力、分销和妆造数据。
