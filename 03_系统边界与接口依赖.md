# 03. 系统边界与接口依赖

版本：v1.0  
日期：2026-05-20

## 1. 一期涉及系统

| 系统 | 定位 | 一期职责 |
| --- | --- | --- |
| BeautyOPC 小程序 | OPC 用户入口和经营工作台 | 入驻、接单、AIGC 内容生成、AIGC 算力、选品、分销、邀请、妆造工作台 |
| BeautyClaw 小程序 | 独立美妆小工具产品 | 一期不承接 BeautyOPC 算力；如需跳转，仅作为后续外部工具入口评估 |
| 商城小程序 | 商品交易与分销系统 | 商品详情、支付、订单、佣金计算、提现打款 |
| 桌面运营后台 | 平台运营管理端 | 审核、任务、通知、AIGC 内容与算力、分销、妆造业务管理 |
| 微信支付 | 支付能力 | AIGC 算力充值、可能的课程支付 |
| 微信/企业微信环境 | 用户沟通和链接转发环境 | 课程链接、问卷链接、面诊链接、总结和推荐链接转发 |

## 2. 系统边界原则

1. BeautyOPC 一期 AIGC 模块只承接图文生成、图片生成、视频生成、视频混剪等内容生产能力。
2. BeautyOPC 不重复开发商城交易、售后、佣金计算和提现打款。
3. BeautyOPC 负责用户经营入口、跨系统跳转、状态展示和业务闭环。
4. 后台负责运营配置和人工处理，不把复杂管理能力堆到小程序端。
5. 一期不深度改造微信聊天页面，只通过小程序卡片、链接和转发完成交互。

## 3. 用户身份依赖

### 3.1 必须确认

- BeautyOPC 与商城是否同主体；BeautyClaw 如后续做跳转再单独确认。
- 是否共用微信开放平台 unionid。
- 小程序间跳转时可携带哪些参数。
- 是否已有统一用户 ID。
- 用户手机号是否必须授权。

### 3.2 建议身份模型

| 字段 | 用途 |
| --- | --- |
| `user_id` | BeautyOPC 内部用户 ID |
| `openid_opc` | BeautyOPC 小程序 openid |
| `openid_claw` | BeautyClaw 小程序 openid，非一期 P0 必需 |
| `openid_mall` | 商城小程序 openid |
| `unionid` | 跨小程序用户统一识别 |
| `opc_id` | OPC 身份 ID |
| `teacher_id` | 妆造师身份 ID |
| `team_id` | 团队 ID |
| `inviter_id` | 邀请人 ID |
| `promoter_id` | 商城推广人 ID |

## 4. AIGC 与运营通用 AI 小工具边界

### 4.1 AIGC 模块

| 接口/能力 | 调用方 | 提供方 | 用途 | 优先级 |
| --- | --- | --- | --- | --- |
| AIGC 任务创建 | BeautyOPC | AIGC 服务 | 创建图文、图片、视频或混剪任务 | P0 |
| AIGC 任务状态 | BeautyOPC | AIGC 服务 | 查询生成进度和结果 | P0 |
| AIGC 算力余额 | BeautyOPC | 算力服务 | 展示用户余额 | P0 |
| AIGC 算力扣减 | BeautyOPC/AIGC 服务 | 算力服务 | 生成成功或按规则扣减 | P0 |
| AIGC 消耗记录 | BeautyOPC | 算力服务 | 展示充值和消耗明细 | P0 |
| AIGC 生成记录 | BeautyOPC | AIGC 服务/素材服务 | 进入素材记录 | P0 |

关键规则：

- AIGC 算力只用于 AIGC 内容生成模块。
- AIGC 算力不与 BeautyClaw 共用。
- 算力扣减必须具备幂等处理。
- AIGC 任务失败时必须明确是否返还算力。
- 生成结果需要明确保存在哪个系统。

### 4.2 运营通用 AI 小工具

AI 测肤、AI 面部分析、AI 妆效生成、AI 推品等作为妆造师运营通用工具提供，不放在 AIGC 模块，不使用 AIGC 算力。

| 工具/能力 | 入口 | 用途 | 一期边界 |
| --- | --- | --- | --- |
| AI 面部分析 | 运营 / 通用工具 | 生成客户专业版与客户版面诊参考 | 可先模板化 + AI 结论 |
| AI 测肤 | 运营 / 通用工具 | 记录肤质、肤色和少量肌肤问题 | 小工具入口，不进入 AIGC |
| AI 妆效生成 | 运营 / 通用工具 | 上传照片生成妆效参考 | 小工具入口，不进入 AIGC |
| AI 推品 | 运营 / 通用工具 | 根据问卷、报告、课程方案匹配商品包 | 小工具入口，不进入 AIGC |

## 5. 商城接口依赖

| 接口/能力 | 调用方 | 提供方 | 用途 | 优先级 |
| --- | --- | --- | --- | --- |
| 商品列表/推荐商品 | BeautyOPC | 商城 | 选品中心展示 | P0 |
| 商品详情跳转路径 | BeautyOPC | 商城 | 进入商品购买页 | P0 |
| 推广人参数识别 | 商城 | BeautyOPC 参数 | 分销归因 | P0 |
| 分销订单查询 | BeautyOPC | 商城 | 展示订单和收益 | P0 |
| 佣金余额查询 | BeautyOPC | 商城 | 展示可提现/累计佣金 | P0 |
| 提现入口或状态 | BeautyOPC | 商城 | 用户查看提现 | P0 |
| 售后/退款状态 | BeautyOPC | 商城 | 影响佣金状态 | P1 |

### 5.1 跳转建议参数

| 参数 | 说明 |
| --- | --- |
| `source=beautyopc` | 来源标识 |
| `promoter_id` | OPC 推广人 ID |
| `teacher_id` | 妆造师 ID |
| `team_id` | 团队 ID |
| `client_id` | 客户 ID，涉及妆造客户归因时使用 |
| `scene` | 使用场景，如选品中心、课程推荐、课后总结 |

### 5.2 商城必须支持的归因结果

- 能识别 BeautyOPC 带来的推广人。
- 能将订单归因到 OPC 或妆造师。
- 能返回分销订单状态。
- 能处理退款后的佣金状态变化。

## 6. 微信支付依赖

| 支付场景 | 是否一期必做 | 说明 |
| --- | --- | --- |
| AIGC 算力充值支付 | 是 | 已确认一期真实充值，且仅用于 BeautyOPC AIGC |
| 妆造课程未付款支付 | 待确认 | 若一期真实试点，建议接入；若仅演示，可先保留支付路径 |
| 商品购买支付 | 由商城负责 | BeautyOPC 不处理商品支付 |
| 邀请奖励提现 | 后置 | 一期先记录奖励状态 |

### 6.1 支付必须确认

- 商户号。
- 支付订单来源系统。
- 支付回调地址。
- 回调幂等规则。
- 退款规则。
- 订单异常处理。

## 7. 企业微信/微信环境依赖

一期建议不做深度消息接管。

必须支持：

- 妆造师从小程序工作台选择课程。
- 通过微信或企业微信转发给客户。
- 客户点击链接进入小程序页面。
- 系统可生成问卷、面诊、总结、推荐链接。

一期不做：

- 修改微信聊天界面。
- 自动读取个人微信消息。
- 客服机器人全托管。
- 企业微信深度会话接管。

后续如转企业微信生态，需要单独评估：

- 企业微信客户联系能力。
- 第三方应用模式。
- 会话存档权限。
- 客户群和客户标签。
- 消息推送限制。

## 8. 运营后台接口依赖

后台需要覆盖以下业务对象：

| 对象 | 后台能力 |
| --- | --- |
| 用户 | 列表、详情、状态、角色 |
| OPC 入驻 | 审核、驳回、审核记录 |
| 任务 | 发布、编辑、上下架、报名人、状态 |
| 通知 | 创建、发送、对象选择、已读状态 |
| 内容页 | 教程、协议、客服、FAQ |
| AIGC 算力 | 套餐、订单、消耗、异常 |
| 分销 | 订单查询、佣金概览、提现状态 |
| 邀请 | 邀请关系、奖励状态 |
| 妆造课程 | 课程、价格、时长、样板 |
| 预约 | 订单、时间、老师、状态 |
| 客户 | 档案、问卷、报告、绑定老师 |
| 老师 | 邀请、新增、修改、解绑 |
| 团队 | 团队看板、客户继承、模板 |

## 9. 文件与素材依赖

需要支持上传和访问：

- OPC 入驻作品或资质。
- 已付款凭证。
- 客户素颜照片。
- AI 面诊报告图。
- 课程总结附件或内容。
- 产品推荐图片。

必须确认：

- 文件存储位置。
- 图片隐私权限。
- 文件大小限制。
- 是否需要图片审核。
- 用户删除和数据留存规则。
