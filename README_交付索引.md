# BeautyOPC 一期开发交付包

版本：v2.0  
日期：2026-05-22  
用途：给业务方做优先级选择，也给开发团队做周期、人力、依赖和验收评估。

## 推荐阅读顺序

1. 打开 `BeautyOPC一期开发评估包_交互版.html` 查看项目目的、主流程和流程卡片。
2. 业务方逐条打开 `flow-pages/` 下的单流程网页，判断是否进入一期。
3. 开发负责人根据已选择流程更新 `04_开发拆分与排期评估.md`。
4. 开发前集中处理 `03_系统边界与接口依赖.md` 和 `05_风险与待确认问题.md`。

## 文件结构

| 文件 | 用途 |
| --- | --- |
| `BeautyOPC一期开发评估包_交互版.html` | 统一展示入口，适合业务汇报和开发评估 |
| `flow-pages/` | 每条流程一份独立网页文档，适合业务方逐条评估 |
| `flows/` | 每条流程一份独立Markdown文档，适合开发拆任务 |
| `01_一期范围冻结说明.md` | 一期范围、优先级建议和边界 |
| `02_功能流程与验收标准.md` | 全流程摘要表 |
| `03_系统边界与接口依赖.md` | 跨系统、接口、权限和数据边界 |
| `04_开发拆分与排期评估.md` | 阶段排期和流程级开发评估 |
| `05_风险与待确认问题.md` | 业务方和外部系统待确认事项 |
| `06_开发任务拆分表.md` | 给开发拆任务用的流程级任务表 |

## 重要原则

- 建议优先级不是最终优先级，最终由业务方决定。
- 每条流程可独立选择进入一期、后置、只做展示占位或暂不开发。
- 估期默认有 AI 协助开发，但仍按真实联调、测试、验收计算。
