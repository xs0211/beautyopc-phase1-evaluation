# BeautyOPC 一期开发评估包

版本：v1.0  
日期：2026-05-20  
用途：用于开发团队评估工作量、排期、人力、接口依赖和一期交付边界。

## 1. 评估包结论

BeautyOPC 一期建议冻结为：

> 通用 OPC 平台底座 + AIGC 内容生成与算力 + 妆造私教样板行业 + 商城/后台基础打通。

一期目标不是做完整生态，而是先完成一个可以试点的经营闭环：

> 用户入驻 → AIGC 创作 → 接单/选品 → 产生收益 → 妆造样板业务验证 → 后台可运营。

| 文件 | 用途 | 建议阅读对象 |

## 3. 推荐使用方式

1. 优先打开 `index.html`，这是统一入口。
2. 向需求方汇报时，使用交互版里的“汇报模式”和 `BeautyOPC一期项目速览_需求方汇报.html`。
3. 开发评估时，在交互版切换到“评估模式”，重点查看范围、接口、排期、风险和任务拆分。
4. 需要二次编辑时，再打开对应 Markdown 文件。
5. 开发前会议逐条处理 `05_风险与待确认问题.md`。

## 4. 一期建议周期

在 AI 协助开发前提下：

- 标准周期：10-12 周。
- 接口成熟、并行充分：8-10 周。
- 支付、商城、AIGC 算力、跨小程序归因不稳定：12 周以上。

## 5. 一期成功标准

一期成功不以“页面数量”判断，而以业务闭环判断：

- 用户可以完成 OPC 入驻并被审核。
- 用户可以进入 AIGC 内容生成、充值 AIGC 算力、查看生成记录。
- 用户可以通过接单广场报名任务。
- 用户可以通过选品中心跳转商城并产生分销归因。
- 妆造师可以完成课程预约、课前问卷、素颜面诊、课程总结和产品推荐。
- 老板可以管理老师、客户归属、团队销售和统一模板。
- 后台可以支撑审核、任务、通知、AIGC 内容与算力、分销和妆造业务运营。
